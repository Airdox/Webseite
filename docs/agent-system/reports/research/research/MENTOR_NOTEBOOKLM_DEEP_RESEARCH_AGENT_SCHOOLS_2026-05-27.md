## Kernfrage

```text
Wie kann ein produktives Multi-Agent-System seine Agenten durch kuratierte Wissensbasen, Agentic RAG, persistenten Memory, Reflexions- und Feedback-Loops, Trainingsszenarien, Evaluationen und Governance nachweisbar intellektuell verbessern, ohne unkontrollierte Selbstveraenderung oder Scheinautonomie zu behaupten?
```

## Deep-Research-Query fuer NotebookLM

```text
Research deeply how production multi-agent AI systems can improve agent knowledge, reasoning quality, and task performance through structured knowledge management, agentic RAG, persistent memory, feedback loops, training scenarios, evaluation rubrics, and governance.

Focus on:
1. multi-agent systems and agent orchestration patterns,
2. agent schools, agent training curricula, agent onboarding, and skill improvement loops,
3. agentic memory: episodic memory, semantic memory, procedural memory, working memory, long-term memory, memory consolidation, learned forgetting,
4. retrieval-augmented generation and agentic RAG for shared team knowledge,
5. knowledge graphs, source-of-truth repositories, runbooks, playbooks, decision logs, and institutional memory,
6. evaluation of agentic systems: task completion, process evaluation, memory recall, policy adherence, tool-use quality, handoff quality, robustness, hallucination control,
7. reflection, self-critique, postmortems, error-to-test loops, human feedback, reinforcement learning from feedback, continual learning, meta-learning,
8. governance: permissions, source freshness, provenance, audit trails, privacy, security, change control, approval gates,
9. practical implementation patterns for a small product team using versioned docs, scripts, reports, and local/NotebookLM knowledge artifacts.

Compare academic papers, engineering guides, benchmark papers, and production architecture guidance. Distinguish clearly between:
- model training or fine-tuning,
- retrieval-based knowledge improvement,
- persistent memory,
- procedural rules and runbooks,
- evaluation and feedback loops,
- governance and safety controls.

Produce:
- a taxonomy of knowledge types for agents,
- a blueprint for an "Agentenschule" or agent academy,
- keyword clusters and source categories,
- recommended source list,
- implementation tasks for the AIRDOX Mentor agent,
- a 30/60/90 day rollout,
- measurable quality indicators,
- risks and safeguards.
```
